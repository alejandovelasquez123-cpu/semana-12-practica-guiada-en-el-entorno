# Checklist de entrega

Antes de subir tu actividad a Moodle, revisa que tengas todo lo siguiente.

## Evidencias obligatorias

- [ ] Bitácora completa.
- [ ] Captura del proyecto abierto en VS Code.
- [ ] Captura del diagnóstico ejecutado.
- [ ] Captura del script principal ejecutado.
- [ ] Captura del reporte generado.
- [ ] Captura del respaldo generado.
- [ ] Captura del dashboard abierto en navegador.
- [ ] Evidencia del caso con errores.
- [x] Taller práctico respondido.
- [x] Conclusión personal.

## Revisión de calidad

- [x] Mis respuestas están escritas con mis propias palabras.
- [x] No entregué solo capturas; también expliqué el proceso.
- [x] Identifiqué entrada, proceso y salida.
- [x] Expliqué la diferencia entre reporte y respaldo.
- [x] Analicé al menos un error o advertencia.
- [x] Mi conclusión relaciona la práctica con un contexto real.

## Archivos sugeridos para entregar

Puedes entregar un solo ZIP con:

```text
bitacora.pdf
capturas/
reporte_generado.txt
respuesta_taller.pdf
```

No subas carpetas innecesarias como `__pycache__` o entornos virtuales.
